Core algorithm

Profile, generate, pair, report.

The benchmark does not ask whether an agent is good at a fixed suite. It asks whether this specific skill changes the outcome on tasks synthesized from its own claimed capability.

Method distinction

Why this is not a fixed task benchmark.

Fixed-suite evaluation

A fixed benchmark can make the skill look useless when tasks are too easy, too hard, or unrelated to the package's real capability. It measures agent performance more than skill contribution.

SkillLens

Each skill produces its own task schemes. The paired condition then isolates the marginal effect of enabling the skill while keeping task content and environment matched.

Utility pass_rate_gain = max(wi_passed - wo_passed, 0) / total_items
Safety security_score = max(10, 100 - sum(base * existence * exploitability))
Cost overhead = wi_effective_tokens - wo_effective_tokens, wi_time - wo_time
