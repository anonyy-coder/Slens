Final Recommendation

The report is not a benchmark-internal accuracy test. It is the adoption summary for this skill.

User question answered: does this skill improve agent performance enough, without unacceptable risk or overhead?
01

Task Gain

Answers the central question: did the target skill improve task completion compared with the matched no-skill baseline?

utility_score = mean((wi_pass - wo_pass) / max(1 - wo_pass, epsilon))
02

Safety

Uses static findings and runtime behavior from skill-enabled runs to flag deployment risk introduced by the skill package.

deduction = base_score * existence_confidence * exploitability
03

Cost

Tracks added or saved execution time and effective input tokens. Cost is reported separately and only breaks ties when utility is indistinguishable.

effective_tokens = input_tokens - cache_tokens

The key claim is paired evaluation: generated tasks are identical across conditions, and the only intended difference is whether the candidate skill is available to the agent.
